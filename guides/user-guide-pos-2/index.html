Introduction

Welcome to jPOS 2.x - A comprehensive Point of Sale and Inventory Management System designed to streamline your business operations. This user guide will walk you through all features and functionalities of the system.

What is jPOS?

jPOS is an enterprise-grade inventory and sales management system that helps businesses manage their products, sales, purchases, transfers, and generate comprehensive reports.

Key Features

  • ‚úì Complete inventory management with stock tracking
  • ‚úì Point of Sale (POS) system with barcode scanning
  • ‚úì Sales and purchase management
  • ‚úì Multi-location stock transfers
  • ‚úì Comprehensive reporting and analytics
  • ‚úì Customer and supplier management
  • ‚úì Expense and income tracking
  • ‚úì Activity logging and audit trails

Installation

Follow these steps to install the JPOS System on your server or localhost environment.

Step 1: Upload the Project

  • Upload the JPOS System project files to your server or localhost directory.
  • Make sure the project root contains the install.php file.

Step 2: Open the Installation Wizard

  1. Open the XAMPP installation folder on your computer.
  2. Go to the htdocs directory.
  3. Copy the provided JPOS System project folder into the htdocs folder.

Example:

C:\xampp\htdocs\jpos-system
  1. Start Apache and MySQL from the XAMPP Control Panel.
  2. Open a web browser (Chrome, Edge, or Firefox).
  3. In the address bar, type the following URL and press Enter:
http://localhost/jpos-system/install.php
  1. The JPOS System Installation Wizard will open in the browser.
Note: Make sure Apache and MySQL services are running in XAMPP before opening the installation page.

Step 3: System Requirements Check

The system will automatically check:

  • PHP version (8.2 or higher)
  • Required PHP extensions
  • Node.js (optional)

Next Steps:

  • If all requirements pass, click "Continue to Database Setup".
  • If any requirement fails, fix it before continuing.

Step 4: Database Configuration

Enter Primary Database details:

  • Database Host (e.g., localhost)
  • Database Name
  • Database Username
  • Database Password

(Optional) Enable Secondary Database:

  • Used for reports and analytics
  • Fill in secondary database details if enabled

Click "Start Installation".

Step 5: Installation Process

During installation, the system will automatically:

  • Create the .env file
  • Generate the application key
  • Run database migrations
  • Seed default users and data
  • Create storage links
  • Set file permissions
  • Optimize the application
  • Create an installation lock file

A progress bar and log messages will be displayed.

Step 6: Installation Complete

  • When installation is finished, a success message will appear.
  • The system will automatically redirect to the Login Page.
For security reasons:
  • The installer file (install.php) is automatically deleted.
  • Re-installation is blocked using a lock file.
üí° Installation Tips:
  • Ensure your server meets all PHP requirements before starting
  • Have your database credentials ready before beginning installation
  • Make sure your web server has write permissions to the project directory
  • If installation fails, check the error logs for detailed information

Goods Received Notes (GRN)

Record all incoming stock with GRN to maintain accurate inventory levels.

Creating a GRN

A Goods Received Note records products received from suppliers and automatically updates inventory stock levels in the system.

  1. Navigate to Stores Goods Received Notes
  2. Click "Create New GRN" or "Add New Goods Received Note"
  3. Fill in the GRN header information:
    • GRN Number: Auto-generated unique identifier
    • Supplier: Select the supplier providing the goods (required)
    • GRN Date: Date goods were received (required)
    • Purchase Order: Link to related POR (optional but recommended)
    • Discount: Apply overall discount if applicable
    • Tax: System calculates tax based on configured rates
    • Remarks: Add notes or special instructions
  4. Add products to the GRN table:
    • If a Purchase Order is linked, products are automatically populated from that PO
    • When a PO is selected, you cannot add new products - only the PO items will appear in the table
    • You can only modify the received quantity for products already in the PO and remove the unnecessary products.
    • Enter received quantity (in purchase units)
    • Verify unit price is correct
    • System calculates line totals automatically
  5. Review all information for accuracy
  6. Click "Create GRN" to finalize and update inventory

GRN Form Overview

Header Section: Contains supplier info, dates, and financial details (discount, tax, totals).

Products Table: Lists received items with quantity, unit price, and line amounts. The system matches received quantities against the linked Purchase Order.

Stock Update: Upon confirmation, all received quantities are added to the relevant inventory, using the purchase unit defined for each product.

Viewing Goods Received Notes (GRN List)

The Goods Received Notes list displays all created GRNs with their details, status, and transaction history.

  1. Navigate to Stores ‚Üí Goods Received Notes
  2. The list displays the following information for each GRN:
    • GRN Number: Unique identifier for each goods receipt
    • Supplier: The supplier who provided the goods
    • Date: Date when goods were received
    • Total Amount: Total value of all products in the receipt
    • Discount: Discount amount applied to the receipt
    • Tax: Tax amount applied to the receipt
    • Status: Processing status of the GRN (Pending, Received, Processed, etc.)
    • Actions: View details, Edit, Print, Delete
  3. Click on any row or the "View" button to see detailed product information and receipts
  4. Use "Delete" to remove GRNs (if they haven't been processed or linked to returns)
‚ö†Ô∏è Important: When a Purchase Order is linked to the GRN, the product list is automatically filled from that PO. You cannot add products that were not in the original PO. This ensures that received goods match the ordered items for proper tracking and reconciliation.
Note: Creating and confirming a GRN automatically updates product stock levels in the system immediately.

GRN Returns

Process returns to suppliers for damaged, defective, or incorrect items received. GRN Returns generate credit notes and reduce inventory accordingly.

Creating a GRN Return

  1. Navigate to Stores ‚Üí GRN Returns
  2. Click "Create New GRN Return" or "Add New Return"
  3. Fill in the return header information:
    • GRN Number: Auto-generated unique identifier
    • Datw: Select the GRN containing the items being returned (required)
  4. Add products to return:
    • Products from the linked GRN appear in the table automatically
    • Enter the Return Quantity for each product being returned (in purchase units)

    • Remove rows for products not being returned

GRN Return Impact

Inventory Update: Returned quantities are deducted from current stock levels.

Credit Note: A credit note is automatically generated for supplier accounting purposes.

Financial Record: The return amount is recorded as a credit against the original purchase.

Note: GRN Returns must be linked to an existing GRN. You cannot create returns for items that were not previously received through a GRN.
üí° Tip: Keep detailed remarks when processing returns to maintain an audit trail for quality issues and supplier accountability.

Taxes

Configure and manage tax rates applicable to your sales transactions. Set up different tax types and rates that will be automatically applied during sales processing.

Viewing Tax List

The tax list displays all configured tax rates with the following information:

  • ID: Unique tax identifier
  • Tax Name: Name of the tax (e.g., "No Tax", "Local Sales Tax", "VAT", "GST")
  • Percentage: Tax rate as a percentage (e.g., 0.00%, 7.00%, 18.00%)
  • Type: Tax calculation method - Inclusive (price includes tax) or Exclusive (tax added to price)
  • Status: Default (system default) or Active status indicator
  • Actions: Edit button to modify tax details

Adding a New Tax

  1. Navigate to Shops ‚Üí Taxes
  2. Click the "Add Tax" button in the top right corner
  3. The "Add New Tax" modal will appear
  4. Fill in the tax details:
    • Tax Name * (required) - Enter a descriptive name for the tax (e.g., "Local Sales Tax", "VAT", "GST", "Import Duty", "Luxury Tax", "Service Tax")
    • Percentage * (required) - Enter the tax rate as a percentage (e.g., 7.00 for 7%, 18.00 for 18%). Default value is 0.
    • Type * (required) - Select tax calculation method from dropdown:
      • Inclusive - Tax is included in the displayed product price (price breakdown shows tax portion)
      • Exclusive - Tax is added on top of the product price (shown as separate line item)
    • Status * (required) - Select tax status from dropdown:
      • Active - Tax can be applied to transactions
      • Inactive - Tax is disabled and cannot be selected
  5. Click "Create Tax" to save the new tax
  6. Click "Cancel" to discard changes

Understanding Tax Types

Inclusive vs Exclusive Taxation:

  • Inclusive Tax: Product price already contains the tax. Example: If an item costs LKR 100 with 10% inclusive tax, the base price is LKR 90.91 and tax is LKR 9.09.
  • Exclusive Tax: Tax is added to the product price. Example: If an item costs LKR 100 with 10% exclusive tax, the final price becomes LKR 110 (LKR 100 + LKR 10 tax).

Editing a Tax

  1. Locate the tax in the tax list
  2. Click the "Edit" button in the Actions column
  3. Update the tax information as needed
  4. Click "Update Tax" to save changes
Note: Fields marked with an asterisk (*) are required. The "No Tax" (ID: 1) is a default system tax with 0.00% rate and cannot be deleted. It's used for tax-exempt transactions.
üí° Tip: Choose "Inclusive" tax for retail scenarios where customers see final prices. Use "Exclusive" tax for B2B transactions where tax is typically added separately. Make sure your tax configuration matches your local tax regulations.

Sales

Process point-of-sale transactions quickly and efficiently using the New Sale / Bill interface. The POS system supports barcode scanning, manual product selection, and multiple payment methods.

Understanding the Sales Interface

The New Sale / Bill screen consists of several key sections:

  • Invoice Number: Auto-generated unique invoice ID (e.g., INV-000001) displayed in top right
  • Scan Barcode: Quick product addition via barcode scanner
  • Customer & Date: Select customer and set transaction date
  • Price Type: Choose between Retail or Wholesale pricing
  • Add Products: Browse and select products from inventory
  • Cart Items: View and manage products in current transaction
  • Bill Summary: Real-time calculation of totals, discounts, and change

Creating a New Sale

  1. Navigate to Shops ‚Üí Sales or click the Dashboard "Sales" tile
  2. The "New Sale / Bill" screen opens automatically
  3. Select Customer & Date:
    • Click the Customer dropdown to select a customer (default shows customer name like "William Taylor")
    • Select the transaction date (defaults to current date)
  4. Choose Price Type:
    • Select Retail for standard customer pricing
    • Select Wholesale for bulk/wholesale pricing
  5. Add Products to Cart: Use one of three methods:
    • Scan Barcode: Type or scan product barcode in the "Scan barcode..." field and click "Add"
    • Browse Products: Click the purple "Browse Products" button to see product catalog
    • Focus Barcode (ESC): Press ESC key to quickly focus barcode input field
  6. Manage Cart Items:
    • Added products appear in the "Cart Items" table showing Product, Price, Qty, Total, and Action columns
    • Adjust quantities as needed
    • Remove items using the Action column buttons
  7. Review Bill Summary:
    • Sub Total: Total before discounts and taxes
    • Custom Discount (LKR): Enter manual discount amount if needed
    • Net Amount: Final amount due
    • Change: Change amount to return (calculated after payment)
  8. Add Payment:
    • Click the "üí≥ Add Payment" button
    • Select payment method (Cash, Card, etc.)
    • Enter amount received
    • System calculates change automatically
  9. Complete Sale:
    • Click "‚úÖ Complete Sale (F9)" button or press F9 key
    • Invoice is generated and saved
    • Receipt can be printed
    • Cart clears for next transaction

Keyboard Shortcuts

Speed up your sales process with these keyboard shortcuts:

  • F9: Complete Sale - Finalize the transaction
  • F8: Clear Cart - Remove all items from cart
  • ESC: Focus Barcode - Jump to barcode input field
  • F3: Complete Sale (alternative)

Shortcuts are displayed at the bottom of the sales screen for quick reference.

Note: Empty cart message displays "No items in cart - Scan barcode or select product to add" when cart is empty. The invoice number is automatically generated for each new transaction and cannot be edited manually.
üí° Tips for Efficient Sales:
  • Use barcode scanning for fastest product entry
  • Set up default customer for walk-in sales to skip customer selection
  • Press F9 to quickly complete sales without using mouse
  • Apply custom discounts before adding payment for accurate change calculation
  • Switch between Retail and Wholesale pricing based on customer type

Sales History

View and manage all completed sales transactions. The Sales History provides a comprehensive record of all sales with detailed information including customer details, products sold, discounts applied, and returns processed.

Understanding Sales History Interface

The Sales History screen displays a comprehensive table with the following columns:

  • #: Sequential row number
  • Invoice No: Unique invoice identifier (e.g., INV-000001)
  • Customer: Customer name associated with the sale
  • Products: List or count of products sold
  • Type: Sale type (Retail, Wholesale, etc.)
  • Total: Original sale total amount
  • Discount: Total discount applied to the sale
  • Net Amount: Amount after discount
  • Returns: Value of returned items
  • Net After Return: Final amount after processing returns
  • Balance: Outstanding balance (for credit sales)
  • Sale Date: Date and time of transaction
  • Actions: Available operations (View, Print, Edit, Delete)

Accessing Sales History

  1. Navigate to Shops ‚Üí Sales History
  2. The sales history table displays all completed transactions
  3. Record count shown in top right corner (e.g., "Total: 0 records")
  4. Use "Back" button to return to previous screen

  1. Locate the desired sale in the list
  2. Click on the row or the "View" action button
  3. View detailed information including:
    • Complete product list with quantities and prices
    • Customer information
    • Payment details and method
    • Applied discounts and taxes
    • Return history (if applicable)

Available Actions

For each sale record, you can perform the following actions:

  • View Details: See complete sale information
  • Print Receipt: Generate and print customer receipt
  • Email Receipt: Send receipt to customer email
  • Edit: Modify sale details (if permitted by permissions)
  • Process Return: Initiate return for items in this sale
  • Void/Cancel: Cancel the sale (if permitted and no returns processed)

Filtering and Searching

Use the available filters to find specific sales:

  • Search by invoice number
  • Filter by customer name
  • Filter by date range
  • Filter by sale type (Retail/Wholesale)
  • Filter by payment status (Paid/Pending/Credit)

Pagination

Navigate through multiple pages of sales records:

  • Results display shows current range (e.g., "Showing 0 to 0 of 0 results")
  • Use "Previous" and "Next" buttons to navigate pages
  • Click page numbers for direct navigation
  • Default display shows 10 records per page
Note: When no sales are found, the table displays "No sales found" message. Sales with returns processed will show the return amount in the Returns column and adjusted final amount in the Net After Return column.
üí° Tip: Use date range filters to view sales for specific periods (daily, weekly, monthly reports). The Balance column helps identify customers with outstanding credit amounts that need collection.

Frequently Asked Questions (FAQ)

Q: How do I change my password?

A: Go to your profile settings (click on your name in the top right corner) and select "Change Password". Enter your current password and new password, then save.

Q: Can I delete a completed sale?

A: Completed sales cannot be deleted to maintain audit trails. However, you can create a return or void the transaction if you have the appropriate permissions.

Q: How do I export data to Excel?

A: Most list views and reports have an "Export" button. Click it and select "Export to Excel" to download the data in Excel format.

Q: What happens when stock reaches minimum level?

A: The system will display an alert on the dashboard and the product will be highlighted in red in the stock reports. You should reorder the product.

Q: Can multiple users work on the system simultaneously?

A: Yes, jPOS supports multiple concurrent users. Each user should have their own login credentials.

Q: How long is data retained in the system?

A: All transaction data is retained indefinitely unless manually deleted by an administrator. Regular backups are recommended.

Q: Can I customize the invoice template?

A: Yes, go to Settings ‚Üí Bill Settings to customize invoice templates, including adding your logo and changing formats.

Q: How do I add a new user?

A: Only administrators can add users. Go to Settings ‚Üí User Management ‚Üí Add New User, then fill in the required details and assign appropriate permissions.
