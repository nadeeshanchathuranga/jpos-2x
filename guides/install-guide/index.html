JAAN System – Installation Manual

1. Introduction

This guide explains how to install the JAAN System (Laravel-based application) using the built-in web installer.

The installer works similarly to WordPress and guides you through system checks, database setup, and application initialization.

2. Server Requirements

Required

  • PHP 8.2 or higher
  • Apache or Nginx Web Server
  • MySQL 5.7+ or MariaDB 10.3+
  • Composer (globally installed)

Required PHP Extensions

  • openssl, pdo, mbstring, tokenizer, xml
  • ctype, json, fileinfo, gd, zip

Optional (Recommended)

  • Node.js (v18+)
  • NPM (frontend build)

3. File & Folder Permissions

storage/
bootstrap/cache/

Linux

chmod -R 775 storage bootstrap/cache
chown -R www-data:www-data .

Windows

Ensure IIS / Apache user has write access.

4. Uploading the Application

  1. Upload all project files to your server root
  2. Ensure these files exist:
install.php
artisan
composer.json
package.json (optional)

Uploaded project files in server root

5. Starting the Installer

https://your-domain.com/install.php
If the application is already installed, the installer will be blocked automatically.

Installer welcome / system check screen

6. Installation Steps

Step 1: System Requirements Check

The installer verifies PHP version, extensions, Composer, and Node.js.

System Requirements Verification Screen

Step 2: Database Configuration

Enter your primary database credentials.

  • Database Host
  • Database Name
  • Database Username
  • Database Password
Optional: Enable Secondary Database only if required.

Primary and Secondary Database Configuration Form

Remote Database (White-Listing Required)

If your database is hosted on a remote server (not localhost), you must whitelist your application server IP.

Steps

  1. Find your server's public IP address
  2. Login to your database server or hosting panel
  3. Add the IP to MySQL allowed hosts / firewall rules

Common Providers

  • cPanel: Remote MySQL → Add Host
  • AWS RDS: Add inbound rule (port 3306)
  • DigitalOcean: Trusted sources → Add IP
If not whitelisted, you will receive a Database Connection Failed error.

Step 3: Admin User Setup

Create the administrator account used to access the system.

Admin User Creation Form

Step 4: Automated Installation

The installer will automatically:

  • Run Composer & NPM
  • Generate .env file
  • Generate APP key
  • Run migrations & seeders
  • Create admin user
  • Build frontend assets
  • Secure the installation

Installation Progress and Live Logs

7. Installation Completion

https://your-domain.com/public
The installer deletes itself and locks the system after completion.

Installation Completed / Redirect Screen

8. After Installation (Recommended)

APP_ENV=production
APP_DEBUG=false
php artisan config:cache
php artisan route:cache

9. Troubleshooting

Composer Not Found

sudo apt install composer

Permission Errors

chmod -R 775 storage bootstrap/cache

Database Connection Failed

  • Verify credentials
  • Ensure database exists
  • Whitelist server IP (remote DB)

10. Security Notes

  • Do not expose .env
  • Do not restore install.php
  • Keep install.lock intact
